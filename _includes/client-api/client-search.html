any( term [, ignore ] )

Match makes which have at least one field that matches the given term.

Parameters:
  • term (String) - A search term to look for. it is applied on the following fields: author, contentType, description, id, remixedFrom, tagPrefix, tags, title, url, user
  • ignore (String or Array) - An optional parameter indicating a field or fields to omit from the any search
Example: {% highlight js %} var makeapi = new Make( optionsObj ); makeapi .any( "foo" ) .then(function( err, makes ) { if ( err ) { // handle error case } // handle success! }); // Don't match makes that only match on "user" makeapi .any( "foo", "user") .then(function( err, makes ) { if ( err ) { // handle error case } // handle success! }); // Don't match makes only that match any of the given fields makeapi .any( "foo",[ "url", "title", "tagPrefix" ]) .then(function( err, makes ) { if ( err ) { // handle error case } // handle success! }); {% endhighlight %}

all( term [, ignore ] )

Match makes that match the term in all fields. best used with the optional ignore parameter, to limit fields from being included in the query.

Parameters:
  • term (String) - A search term to look for. it is applied on the following fields: author, contentType, description, id, remixedFrom, tagPrefix, tags, title, url, user
  • ignore (String or Array) - An optional parameter indicating a field or fields to omit from the any search
Example: {% highlight js %} var makeapi = new Make( optionsObj ); // match makes that contain foo in all fields (basically useless without the `ignore` param) makeapi .all( "foo" ) .then(function( err, makes ) { if ( err ) { // handle error case } // handle success! }); // Don't match on the given field makeapi .all( "foo", "user") .then(function( err, makes ) { if ( err ) { // handle error case } // handle success! }); // exclude the given fields from the query makeapi .all( "foo",[ "url", "title", "tagPrefix" ]) .then(function( err, makes ) { if ( err ) { // handle error case } // handle success! }); {% endhighlight %}

sortByField( field [, direction ] )

Sort the makes matched By Elastic Search using the specified field and direction. Can be called multiple times to add more than one sort field, with the first fields taking precedence.

Parameters:
  • field (String) - A String matching one valid field of a make. i.e. "createdAt"
  • direction (String) - Either "asc" or "desc", specifying ascending or descending sort, respectively. defaults to "asc"
Example: {% highlight js %} var makeapi = new Make( options ); makeapi .sortByField( "updatedAt" ) .then(function( err, makes ) {}); makeapi .sortByField( "createdAt", "desc" ) .sortByField( "updatedAt", "asc" ) .then(function( err, makes ) {}); {% endhighlight %}
{% include anchor.html id="tags" %} {% include top.html %}

tags( options [, not ] )

Add a filter to the search for the specified tag[s]

Parameters:
  • options (Object || Array) - An Object or Array. If it is an Array, it should contain one or all of the tags you wish to filter against. If it is an object, it must contain the following:
    • tags (Array) - An Array of tags, specified as strings
    • execution (String) - can be one of "and" and "or" Specifies if multiple tags should be searched for using AND/OR logic. defaults to "and". I.E. "tag1" OR "tag2"
    • **YOU MUST USE THE OBJECT STYLE DEFINITION WITH find
  • not (Boolean) - if true, only makes that do not match the given tag[s]
Example: {% highlight js %} var makeapi = new Make( options ); makeapi .tags( [ "foo", "bar", "baz" ] ) .then(function( err, makes ) {}); makeapi .tags({ tags: [ "foo", "bar", "baz" ] }) .then(function( err, makes ) {}); makeapi .tags({ tags: [ "foo", "bar", "baz" ], execution: "or" // defaults to "and" }) .then(function( err, makes ) {}); {% endhighlight %}
